## 1. Overview

### 1.1 Design Philosophy

ROM Runner's UI follows a modern desktop application paradigm, drawing inspiration from applications like Steam, Plex, and iTunes. The interface prioritizes:

- **Discoverability**: Users should quickly understand available actions
- **Efficiency**: Common workflows require minimal clicks
- **Clarity**: Visual hierarchy guides attention to important information
- **Responsiveness**: UI responds immediately to user input with appropriate feedback

### 1.2 Core User Workflows

| Workflow | Primary Goal | Key Screens |
|----------|-------------|-------------|
| **Import** | Add ROMs to library | Library â†’ Add Folder Dialog |
| **Browse** | Find and explore games | Library â†’ Game Details |
| **Organize** | Create collections | Library â†’ Collection Management |
| **Deploy** | Transfer to device | Device Setup â†’ Deployment Config â†’ Progress |
| **Configure** | Manage settings | Settings â†’ BIOS Manager |

### 1.3 Screen Types

| Type | Behavior | Examples |
|------|----------|----------|
| **Primary Screens** | Full workspace, sidebar visible | Library, Devices, Settings |
| **Modal Dialogs** | Overlay, blocks background | Game Details, Add to Collection |
| **Wizards** | Multi-step flow with progress | Device Setup Wizard |
| **Panels** | Slide-in from edge | Quick Filters, Notifications |

## 8. Theme System *(NEW in v1.1.0)*

### 8.1 Overview

ROM Runner supports multiple visual themes beyond basic light/dark mode. The theme system allows users to customize the application's appearance while maintaining consistent functionality.

**Reference:** See `docs/specs/ROM_Runner_Theme_System_Specification_v1_0_0.md` for complete theme definitions.

### 8.2 Built-in Theme Families

| Theme Family | Description | Light Mode | Dark Mode |
|--------------|-------------|:----------:|:---------:|
| **Default** | Clean, modern professional | âœ“ | âœ“ |
| **Neumorphism** | Soft shadows, extruded UI | âœ“ | âœ“ |
| **Retro** | Vintage gaming aesthetic | âœ“ | âœ“ |
| **Terminal** | Green phosphor on black | â€” | Fixed |

### 8.3 Theme Application

Themes are applied via CSS classes on the document root:


### 8.4 ThemeProvider Component

```typescript
// components/providers/ThemeProvider.tsx
interface ThemeProviderProps {
  children: React.ReactNode;
}

const ThemeProvider: React.FC<ThemeProviderProps> = ({ children }) => {
  const themeFamily = useSettingsStore((s) => s.themeFamily);
  const themeMode = useSettingsStore((s) => s.themeMode);
  const applyTheme = useSettingsStore((s) => s.applyTheme);
  
  // Apply theme on mount and changes
  useEffect(() => {
    applyTheme();
  }, [themeFamily, themeMode, applyTheme]);
  
  // Listen for system theme changes
  useEffect(() => {
    if (themeMode !== 'system') return;
    
    const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)');
    const handler = () => applyTheme();
    mediaQuery.addEventListener('change', handler);
    return () => mediaQuery.removeEventListener('change', handler);
  }, [themeMode, applyTheme]);
  
  return <>{children}</>;
};
```

### 8.5 Theme-Aware Components

Components should use CSS custom properties for theme-aware styling:

```tsx

// âœ“ Use CSS custom properties

// âœ“ Or use Tailwind's theme-aware utilities

```

### 8.6 Neumorphic Components

For the neumorphic theme, components should adapt their shadows:

```tsx
// Neumorphic-aware button

  <button
    className={cn(
      'rounded-lg px-4 py-2 transition-shadow',

      'hover:shadow-neu-flat'
    )}
    {...props}
  >
    {children}
  </button>
);
```

### 8.7 Terminal Theme Specifics

The terminal theme has unique requirements:

- Always uses dark mode (ignores themeMode)
- Monospace font for all text
- Green phosphor color (`#00ff00`) for primary elements
- Optional scanline overlay effect
- CRT glow effects on interactive elements

```tsx
// Terminal-aware text
<span className="theme-terminal:font-mono theme-terminal:text-[var(--glow-color)]">
  {text}
</span>
```
